ZXPairing - 开发执行路线图

执行策略： 后端先行 (Core) -> 移动端突击 (Mobile) -> PC端补全 (Web) -> 智能化 (AI)

阶段一：心脏构建 (The Core) - MVP V0.5a

目标： 确保“算法”是准的，确保“接口”是通的。不涉及任何界面开发。

1. 算法引擎封装/多组任务： 编写 Python 脚本，封装 bbpPairings (C++) 和内部 RoundRobin 逻辑。

核心动作： 输入选手List -> 输出对阵JSON。

测试： 用 10人单循环、11人瑞士制跑 1000 次，确保无报错、无死锁。

2. 基础后端 (Node.js/Python)

任务： 搭建 API 服务。

核心接口： POST /match/pair (调用算法), POST /match/result (录入成绩)。

验证： 使用 Postman 工具调通所有接口。

阶段二：移动端突击 (Mobile First) - V1.0：

目标： 解决“极速切磋”场景，让教练能把手机拿在手里用。

1. 小程序极速模式

任务： 开发“粘贴名单”页、“对阵列表”页、“录分弹窗”。

技术： 微信原生开发 或 Uni-app。

本地化： 仅做基本缓存，需联网使用。验证： 真机调试，从粘贴名单到生成对阵耗时 < 5秒。

2. 基础分享与榜单

任务： 利用 Canvas 生成排名图片。

验证： 发到微信群，点击图片清晰度达标。

阶段三：SaaS 化与 PC 端 (Web Admin) - V1.

目标： 解决“正式比赛”场景，支持报名、收费、大屏。

1. 账号与数据打通

任务： 引入 MongoDB/MySQL 数据库，实现数据持久化。

任务： 实现“微信登录”，打通 OpenID。

2. PC Web 管理后台

任务： 开发 React/Vue 网页版。

功能： 扫码登录、Excel 批量粘贴导入、大屏投射视图 (WebSocket)。照场景验证： 电脑上操作，手机上能实时看到数据变化。 �⚠️阶段四：AI 与 商业化 (V2.0)量

目标： 提升壁垒，变现。

1. 权限与支付

任务： 对接微信支付，开发 RBAC 权限中间件。

2. AI OCR

任务： 对接通用文字识别 API (腾讯云/阿里云)。步动. 荣誉体系

积分榜: 支持 团体总分 (自动计算同单位前3名总分)。

等级分: 实时计算 Elo 变动 (+4.2)。

分享: 手机端一键生成“战报长图”。

3. 商业化与会员体系 (Monetization)

免费版 (Free): 仅极速/联赛模式，单人操作，<30人。

专业版 (Pro):

付费策略: 单场解锁 (￥9.9) 或 年费会员 (￥299)。

权益: 解锁 AI 录入、团体分、导出 PDF、多人协作。
